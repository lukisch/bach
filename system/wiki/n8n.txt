N8N - WORKFLOW-AUTOMATISIERUNG
==============================

WAS IST N8N?
------------
n8n (ausgesprochen "n-eight-n") ist eine Open-Source Workflow-
Automatisierungsplattform (Low-Code/No-Code). Vergleichbar mit
Zapier oder Make, aber selbst gehostet.

KERNKONZEPT
-----------
  Trigger -> Node -> Node -> Node -> Aktion
     |         |        |         |
  Email   ->  PDF   -> LLM   -> Speichern
  kommt      lesen    fragen    in DB

HOSTING-OPTIONEN
----------------
- Self-hosted: Docker Container auf eigenem Server
- n8n Cloud: Gehostete Version (ab ~20 EUR/Monat)

VERGLEICH
---------
  Aspekt        n8n              Zapier/Make      Python-Script
  ---------------------------------------------------------------
  Hosting       Self/Cloud       Nur Cloud        Lokal
  Kosten        Kostenlos*       Ab 20 EUR/M      Kostenlos
  Nodes         400+             5000+            Libraries
  GUI           Visuell          Visuell          Code
  Kontrolle     Mittel           Gering           Voll

  *Self-hosted Version ist kostenlos (Fair-Code Lizenz)

WANN N8N NUTZEN?
----------------
Gut fuer:
- Cloud-Dienste anbinden (Gmail, Drive, Slack, etc.)
- 24/7 Ueberwachung (Trigger-basiert)
- Einfache Workflows ohne Programmierung
- API-Aggregation aus verschiedenen Quellen

Weniger geeignet fuer:
- Komplexe Logik mit vielen Bedingungen
- Volle Kontrolle ueber Ausfuehrung
- Offline-Betrieb
- Datenschutz-kritische Daten

TYPISCHE EINSATZGEBIETE
-----------------------
- Email-Anhaenge automatisch verarbeiten
- Google Drive / Dropbox ueberwachen
- Social Media Posts automatisieren
- CRM-Daten synchronisieren
- Benachrichtigungen senden

N8N + BACH HYBRID
-----------------
Sinnvolle Arbeitsteilung:

  n8n (Postbote):
  - Emails mit Rechnungen abholen
  - PDFs aus Cloud-Speicher holen
  - In lokalen Ordner ablegen
  - Webhook an BACH senden

  BACH (Sachbearbeiter):
  - Intelligente Klassifizierung
  - Entscheidungen mit Rueckfragen
  - Komplexe Faelle bearbeiten
  - Finale Kontrolle beim User

BEISPIEL-WORKFLOW
-----------------
Der analysierte "KI-Buchhalter" Workflow:

  1. Google Drive Trigger (neue PDF)
  2. PDF herunterladen
  3. Text aus PDF extrahieren
  4. LLM extrahiert: Firma, Betrag, Datum, etc.
  5. Ordner Jahr/Monat erstellen
  6. Datei verschieben + umbenennen
  7. Google Sheets aktualisieren
