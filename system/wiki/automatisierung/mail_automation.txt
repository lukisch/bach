EMAIL-KATEGORISIERUNG
=====================
  LLMs koennen Mails automatisch klassifizieren:
    - Support-Anfrage
    - Spam
    - Lead/Verkaufschance
    - Intern/Extern
    - Dringlichkeit (Hoch/Mittel/Niedrig)

  Workflow:
    Mail → LLM klassifiziert → Routing → Entwurf → Review

BACH-INTEGRATION
================
  Empfohlener Workflow:
    1. Email-Abruf via IMAP/POP3 oder Gmail API
    2. Klassifizierung durch LLM
    3. RAG fuer kontextbasierte Antwort-Generierung
    4. Menschliche Freigabe fuer wichtige Mails

  Partner-Zuweisung (BACH-spezifisch):
    - Ollama: Email-Entwuerfe (10+ pro Tag) - Datenschutz!
    - Claude: Komplexe, nuancierte Antworten
    - Gemini: Hohe Volumina, Kosteneffizienz

  Vorteil Ollama fuer Mails:
    - Keine Mail-Inhalte verlassen lokales System
    - DSGVO-konform
    - Personenbezogene Daten bleiben lokal

  Tools-Integration:
    - tools/email_classifier.py (zu erstellen)
    - tools/email_responder.py (zu erstellen)

N8N WORKFLOW BEISPIEL
=====================
  1. Trigger: Neue Email
  2. n8n ruft Ollama API (lokal)
  3. Ollama klassifiziert und entwirft
  4. n8n sendet Entwurf zur Freigabe
  5. Nach Freigabe: Automatischer Versand

SICHERHEITSHINWEISE
===================
  - Sensible Daten niemals an Cloud-LLMs
  - Email-Credentials sicher speichern
  - Keine automatischen Antworten ohne Review
  - DSGVO: Ollama fuer EU-Kundendaten
