WAS IST VISUALISIERUNG?
=======================

Visualisierung ist die Darstellung von Informationen, Ideen
und Zusammenhaengen in visueller Form. Sie macht Abstraktes
greifbar und foerdert Verstaendnis und Erinnerung.

  Warum visualisieren?
  --------------------
  - 65% der Menschen sind visuelle Lerner
  - Bilder werden 60.000x schneller verarbeitet als Text
  - Visuelle Information bleibt laenger im Gedaechtnis
  - Komplexe Zusammenhaenge werden verstaendlicher
  - Gruppen bekommen ein gemeinsames Bild

  Grundprinzip
  ------------
  "Ein Bild sagt mehr als tausend Worte"
  - aber nur, wenn es das RICHTIGE Bild ist!

GRUNDLAGEN VISUELLEN DENKENS
============================

  Visual Alphabet
  ---------------
  Alles laesst sich aus 5 Grundformen zusammensetzen:

    O     Kreis      -> Koepfe, Sonne, Ideen
    []    Rechteck   -> Container, Gebaeude
    /\    Dreieck    -> Richtung, Hierarchie
    |     Linie      -> Verbindung, Trennung
    .     Punkt      -> Aufzaehlung, Betonung

  Daraus entstehen:
    O + | = Strichmaennchen
    [] + /\ = Haus
    O + [] = Sprechblase

  Visuelle Grammatik
  ------------------
  - Container: Rahmen = Zusammengehoerigkeit
  - Pfeile: Verbindung = Beziehung/Fluss
  - Groesse: Gross = Wichtig
  - Farbe: Hervorhebung = Unterscheidung
  - Position: Oben = Wichtiger/Frueher

  Gestaltgesetze nutzen
  ---------------------
  - Naehe: Zusammen = Zusammengehoerig
  - Aehnlichkeit: Gleich = Kategorie
  - Geschlossenheit: Umrandet = Einheit
  - Kontinuitaet: Linie = Zusammenhang
  - Figur/Grund: Kontrast = Fokus

ANALOGE VISUALISIERUNG
======================

  Flipchart-Gestaltung
  --------------------
  Das Flipchart ist das Arbeitspferd der Visualisierung.

  Grundaufbau:
  +-------------------------------+
  |    TITEL (gross, lesbar)      |
  +-------------------------------+
  |                               |
  |    Inhalt                     |
  |    * Punkt 1                  |
  |    * Punkt 2                  |
  |    * Punkt 3                  |
  |                               |
  |         [Bild]                |
  |                               |
  +-------------------------------+

  Tipps:
  - Titel IMMER in Druckbuchstaben
  - Maximal 7 Punkte pro Blatt
  - 1/3 Text, 2/3 Weissraum
  - Farben sparsam (3-4 maximal)
  - Dicke Marker fuer Titel, duenne fuer Text

  Farbsystem
  ----------
  - Schwarz/Dunkelblau: Haupttext
  - Rot: Hervorhebung, Wichtig
  - Gruen: Positiv, Erledigt
  - Orange/Gelb: Markierung, Achtung
  - Grau: Schatten, Hintergrund

  Moderationswand
  ---------------
  Grosse Flaeche fuer Kartenarbeit.

  Strukturierungsmoeglichkeiten:
  - Matrix (2x2, 3x3)
  - Timeline (Zeitstrahl)
  - Cluster (Themengruppen)
  - Ranking (Priorisierung)
  - Flussdiagramm

SKETCH NOTES (Visuelle Notizen)
===============================

  Was sind Sketch Notes?
  ----------------------
  Handschriftliche Notizen, angereichert mit visuellen
  Elementen. Kombination aus Text, Icons und Strukturen.

  Vorteile:
  - Aktives Zuhoeren (man muss filtern)
  - Bessere Erinnerung
  - Einzigartig und persoenlich
  - Macht Spass

  Grundelemente
  -------------

  Text-Hierarchie:
    TITEL     <- Blockschrift, gross
    Untertitel <- Druckschrift, mittel
    FlieÃŸtext  <- Handschrift, klein

  Container:
    +--------+   [......]   (      )   <      >
    |  Box   |   :Banner:   (Wolke)    <Pfeil>
    +--------+   '......'   (      )   <      >

  Verbinder:
    ->  Pfeil (Richtung)
    --  Linie (Verbindung)
    ... Punkte (Fortsetzung)
    === Doppellinie (Trennung)

  Icons (einfache Symbole):
    ! Ausruf    ? Frage     * Idee
    O Person    [] Dokument /\ Berg (Ziel)
    -> Prozess  <3 Herz     $ Geld

  Layout-Muster
  -------------

  Linear (klassisch):
    1. Punkt
       - Detail
    2. Punkt
       - Detail

  Radial (Mind Map):
           Thema A
              |
    Thema B --+-- Thema C
              |
           ZENTRUM

  Spalten (Vergleich):
    | Pro     | Contra  |
    |---------|---------|
    | + Gut   | - Schlecht |

  Pfad (Story):
    [Start] -> [Mitte] -> [Ende]

MIND MAPPING
============

  Grundprinzip
  ------------
  Zentrale Idee in der Mitte, Aesteentwicklung nach aussen.
  Nutzt beide Gehirnhaelften (Struktur + Bild).

  Erstellung
  ----------
  1. Zentrales Thema in die Mitte (mit Bild)
  2. Hauptaeste (Hauptthemen) nach aussen
  3. Nebenaeste (Details) an Hauptaeste
  4. Keywords statt Saetze
  5. Farben und Bilder ergaenzen

  Beispiel:
                    [Finanzen]
                        |
        [Marketing]---[PROJEKT]---[Technik]
                        |
                    [Personal]

  Regeln (nach Buzan)
  -------------------
  - Querformat verwenden
  - Organische, gebogene Linien
  - Ein Wort pro Ast
  - Bilder und Symbole verwenden
  - Farben unterscheiden Aeste
  - Aeussere Aeste duenner als innere

  Anwendungen
  -----------
  - Brainstorming
  - Notizen in Meetings
  - Lernstoff strukturieren
  - Reden vorbereiten
  - Projektplanung

INFOGRAFIKEN
============

  Definition
  ----------
  Professionelle visuelle Darstellung von Daten und
  Informationen fuer Kommunikation und Publikation.

  Typen
  -----
  - Statistische (Diagramme, Charts)
  - Prozesse (Ablaufdiagramme)
  - Geographische (Karten)
  - Vergleiche (Gegenuberstellungen)
  - Hierarchien (Organigramme)
  - Timeline (Zeitstraehle)

  Diagrammtypen
  -------------
  Balken/Saeulen: Vergleich von Kategorien
  Linien: Entwicklung ueber Zeit
  Kreis/Torte: Anteile eines Ganzen
  Scatter: Korrelation zweier Variablen
  Flaechen: Kumulierte Entwicklung

  Gestaltungsprinzipien
  ---------------------
  - Daten nicht verzerren (y-Achse bei 0 beginnen)
  - Einfachheit (Chartjunk vermeiden)
  - Beschriftung lesbar
  - Farben konsistent
  - Quelle angeben

GRAPHIC RECORDING
=================

  Was ist das?
  ------------
  Live-Visualisierung von Veranstaltungen auf grossen
  Flaechen (Papierrolle, digitale Wand).

  Ablauf
  ------
  1. Vorbereitung: Template mit Struktur
  2. Waehrend: Kernaussagen einfangen
  3. Verdichtung: Zusammenhaenge zeigen
  4. Nachbereitung: Foto/Scan verteilen

  Faehigkeiten
  ------------
  - Schnell zeichnen
  - Zuhoeren und filtern
  - Raeumlich denken
  - Improvisation

  Unterschied zu Sketch Notes
  ---------------------------
  - Gross statt klein
  - Fuer Publikum statt fuer sich
  - Live sichtbar
  - Professioneller Kontext

BACH-INTEGRATION
================

  Visualisierung in BACH
  ----------------------
  - GUI Dashboard als Visualisierung
  - Memory als Mind Map denkbar
  - Task-Board als Kanban

  Dokumentation
  -------------
  - Diagramme in docs/ speichern
  - ARCHITECTURE_DIAGRAMS.md als Beispiel
  - ASCII-Art fuer Help-Dateien

  ASCII-Visualisierung
  --------------------
  In txt-Dateien moeglich:

    +----------+     +----------+
    |  Input   | --> | Process  | --> Output
    +----------+     +----------+

  Einschraenkungen
  ----------------
  - Keine Farben in txt
  - Begrenzte Zeichen
  - Monospace-Schrift noetig
