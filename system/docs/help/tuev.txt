TUEV-HANDLER
============

Workflow-Qualitaetssicherung und Testfall-Management fuer BACH-Workflows.
Der TUeV-Handler verwaltet den Status, die Gueltigkeit und die Qualitaetsmetriken
von Workflows ueber ihre Lebenszyklen.


HANDLER-NAME
============

bach tuev     - Workflow-TUeV und Gueltigkeit
bach usecase  - Testfall-Definition und Ausfuehrung


BESCHREIBUNG
============

Der TUeV-Handler (Test, Ueberpruefen, Validierung) verwaltet:

  * Registrierung von Workflows in der Bach-Datenbank
  * Tracking der TUeV-Gueltigkeit (Standard: 90 Tage)
  * Uebersicht ueber abgelaufene/bald faellige Workflows
  * Verwaltung von Testfaellen (Usecases) pro Workflow
  * Status und Qualitaets-Score pro Testfall

Der Handler arbeitet mit:
  * Datenbank: data/bach.db (workflow_tuev, usecases Tabellen)
  * Workflows: skills/_workflows/ (*.md Dateien)
  * Test-Input/Output als JSON in DB


OPERATIONEN
===========

TUEV-OPERATIONEN:

  bach tuev
  bach tuev status        - Status aller Workflows anzeigen
  bach tuev check WFNAME  - Einzelnen Workflow und seine Usecases pruefen
  bach tuev run           - Alle abgelaufenen Workflows auflisten
  bach tuev renew WFNAME  - TUeV erneuern (90 Tage Gueltigkeit)
  bach tuev init          - Workflows aus skills/_workflows/ registrieren

USECASE-OPERATIONEN:

  bach usecase list [WF]       - Alle oder gefilterte Testfaelle anzeigen
  bach usecase add WFNAME      - Neuen Testfall hinzufuegen (SQL-Template)
  bach usecase show ID         - Testfall-Details mit Input/Output
  bach usecase run ID          - Testfall ausfuehren und pruefen
  bach usecase run-all WFNAME  - Alle Testfaelle eines Workflows ausfuehren


BEISPIELE
=========

1. Neue Workflows in DB aufnehmen:
   $ bach tuev init
   [TUeV] Workflows initialisiert
     Hinzugefuegt: 5
     Uebersprungen: 0

2. Status-Uebersicht (sortiert nach Ablaufdatum):
   $ bach tuev status
   [WORKFLOW-TUeV] Status-Uebersicht

   ABGELAUFEN (sofort pruefen!):
     [!] bugfix-protokoll           Score: 75% | Status: pending

   BALD FAELLIG (< 14 Tage):
     [~] data-pipeline              Score: 88% | 5d

   OK:
     [OK] main-workflow             Score: 92% | 30d

3. Workflow-Pruefung mit Usecases:
   $ bach tuev check bugfix-protokoll
   [TUeV] Pruefung: bugfix-protokoll
   Pfad: skills/_workflows/bugfix-protokoll.md
   Status: pending
   Score: 75%

   Usecases: 2
     [?] Einfacher Testfall (?)
     [OK] Edge-Case mit Fehler (pass)

4. TUeV erneuern nach erfolgreicher Pruefung:
   $ bach tuev renew bugfix-protokoll
   [TUeV] bugfix-protokoll erneuert
     Gueltig bis: 2026-06-02

5. Neuen Testfall hinzufuegen:
   $ bach usecase add data-pipeline
   SQL zum Einfuegen:
   ---------------------
   INSERT INTO usecases (title, description, workflow_name, ...)

   Oder via GUI: /usecases

   Beispiel:
   $ bach db query "INSERT INTO usecases (...) VALUES (...)"

6. Testfall ausfuehren:
   $ bach usecase run 5
   [USECASE] Test #5: Einfacher Test
   [INFO] Manuelle Pruefung erforderlich.

   Ergebnis eintragen:
   $ bach db query "UPDATE usecases SET test_result='pass', test_score=90 WHERE id=5"
